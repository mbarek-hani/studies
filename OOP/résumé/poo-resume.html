Résumé du Cours de Programmation Orientée Objet

Introduction à la POO

La Programmation Orientée Objet est un paradigme de programmation basé sur le concept d'objets, qui peuvent contenir des données et du code. Les principes fondamentaux incluent l'encapsulation, l'héritage, et le polymorphisme.

Concepts clés

Objet Classe Encapsulation Héritage Polymorphisme Abstraction

Chapitre 3: Créer un Objet

Définition d'un Objet

Un objet est composé de trois éléments essentiels:

OBJET = Référent + État + Comportement
  • Référent: Nom qui identifie l'objet et contient l'adresse mémoire où l'objet est stocké
  • État: Ensemble d'informations (attributs) décrivant l'objet
  • Comportement: Ensemble de méthodes qui peuvent modifier l'état de l'objet

Exemple: Une voiture peut avoir comme attributs couleur="rouge", marque="Peugeot" et comme comportements démarrer(), accélérer(), etc.

Adressage Indirect

Plusieurs référents peuvent pointer vers un même objet en mémoire. La modification de l'objet via un référent sera visible via l'autre référent.

Objet Composite

Un objet peut contenir d'autres objets comme attributs. Par exemple, une voiture peut contenir un objet moteur.

Instanciation

L'instanciation est le processus de création d'un objet à partir d'une classe. Elle comporte deux phases:

  1. Création de l'objet en mémoire (ressort de la classe)
  2. Initialisation des attributs (ressort de l'objet)

L'instanciation explicite se fait via un constructeur, appelé automatiquement lors de la création de l'objet.

Destruction d'un Objet

La destruction explicite d'un objet s'effectue en appelant le destructeur de la classe. Le destructeur est une méthode particulière qui permet la libération des ressources d'un objet non référencé.

Chapitre 4: L'Encapsulation

Principe de l'encapsulation

L'encapsulation consiste à:

  • Réunir le code (méthodes) et les données (attributs) dans une même entité
  • Protéger l'information contenue dans un objet
  • Masquer les détails d'implémentation en définissant une interface

L'interface est la vue externe d'un objet, définissant les services accessibles (méthodes publiques) aux utilisateurs de l'objet.

Exemple: Dans une classe Compte, les attributs (numéro, solde) sont privés et ne sont accessibles que via des méthodes publiques comme retirer() et déposer().

Modificateurs et Accesseurs

Pour interagir avec les attributs privés d'un objet:

  • Accesseurs (getters): Méthodes qui retournent la valeur d'un attribut (getCouleur())
  • Modificateurs (setters): Méthodes qui modifient la valeur d'un attribut (setCouleur("bleu"))

Chapitre 5: Manipuler les Méthodes

Définition d'une Méthode

Une méthode est composée:

  • D'une entête: nom, type de retour, paramètres
  • D'un bloc d'instructions (corps de la méthode)
Entier CalculSomme(x: entier, y: entier)
{
  z: entier
  z = x + y
  retourner z
}

Visibilité d'une Méthode

Visibilité Symbole Description
Publique + Accessible par toutes les classes
Protégée # Accessible par la classe elle-même et ses classes héritières
Privée - Accessible uniquement par la classe elle-même

Appel d'une Méthode

L'appel d'une méthode d'instance se fait en nommant l'objet suivi d'un point et du nom de la méthode:

nomObjet.nomMethode(arg1, arg2,...)

Exemple: voiture.demarrer()

Méthodes de Classe

Une méthode de classe est liée à la classe et non à ses instances. Son exécution ne dépend pas des objets.

Appel d'une méthode de classe: nomClasse.nomMethode(arg1, arg2,...)

Important: Les méthodes de classe ne peuvent pas accéder aux attributs d'instance.

  • Méthodes d'instance → peuvent accéder aux attributs d'instance, méthodes d'instance, attributs de classe, méthodes de classe
  • Méthodes de classe → peuvent accéder aux attributs de classe et méthodes de classe uniquement

Chapitre 1 (Partie 2): L'Héritage

Principe de l'Héritage

L'héritage définit une relation de spécialisation ou de généralisation entre différentes classes.

Quand une classe A hérite d'une classe B:

  • A acquiert toutes les propriétés de B, plus ses propres propriétés
  • A est une spécialisation de B; B est une généralisation de A
  • A est la classe fille (dérivée); B est la classe mère (super-classe)
  • Toute instance de A peut être considérée comme un objet de B

Exemple: La classe Etudiant hérite de la classe Personne, ajoutant des attributs spécifiques comme note1 et note2.

L'héritage permet de:

  • Factoriser le code en regroupant les caractéristiques communes
  • Faciliter la création de nouvelles classes grâce à une hiérarchie bien définie

Types d'Héritage

  • Héritage multiple: Une classe peut hériter de plusieurs classes (ex: Etudiant hérite de Personne et EtreVivant)
  • Héritage en cascade: Une sous-classe peut être elle-même une super-classe (ex: EtudiantDeuxiemeCycle hérite de Etudiant qui hérite de Personne)

Surcharge des Constructeurs et Chaînage

  • Surcharge du constructeur: Plusieurs constructeurs dans une même classe avec différentes signatures (nombre/types de paramètres)
  • Chaînage des constructeurs: Appel d'un constructeur de la classe mère depuis un constructeur de la classe fille pour éviter la duplication de code

Exemple de chaînage:

CréerEtudiant(nom: chaine, CIN: entier, dateNaiss: entier, note1: réel, note2: réel)
{
  CréerPersonne(nom, CIN, dateNaiss); // Appel du constructeur de la classe mère
  this.note1 = note1;
  this.note2 = note2;
}

Visibilité des Membres dans l'Héritage

  • Les membres privés (-) de la classe mère ne sont pas accessibles par les classes filles
  • Les membres protégés (#) sont accessibles par les classes filles
  • Les membres publics (+) sont accessibles par toutes les classes

Pour permettre à une classe fille d'accéder à un attribut de la classe mère, celui-ci doit être déclaré comme protégé (#) et non privé (-).

Conclusion

La Programmation Orientée Objet offre une approche structurée pour organiser le code en entités réutilisables (objets) qui encapsulent données et comportements. Les concepts d'encapsulation, d'héritage et de polymorphisme permettent de créer des architectures de code flexibles et maintenables.
