Programmation Orientée Objet - Résumé

Chapitre 3 : Créer un objet

Définition d'un objet

Un objet = Référent + État + Comportement

  • Référent : identifiant unique (adresse mémoire)
  • État : valeurs des attributs à un instant donné
  • Comportement : méthodes qui modifient l'état

Exemple d'objet composite :

Voiture { Moteur; // Objet composant Couleur: "rouge"; }

Instanciation

  • 2 phases : allocation mémoire + initialisation attributs
  • Rôle du constructeur

Destruction d'objet

Via le destructeur quand l'objet n'est plus référencé

Chapitre 4 : Encapsulation

Principe

  • Protection des données internes
  • Interface publique pour l'interaction
  • Utilisation des getters/setters
class Compte { private solde; public deposer() { ... } }

Chapitre 5 : Manipuler les méthodes

Caractéristiques

  • Visibilité (public, protected, private)
  • Paramètres et valeur de retour
  • Différence méthodes de classe vs d'instance

Héritage

Principe

  • Relation parent-enfant entre classes
  • Réutilisation du code
  • Exemple : Etudiant hérite de Personne

Types

  • Simple : une classe mère
  • Multiple : plusieurs classes mères
  • En cascade : hiérarchie multi-niveaux
